# Iron Chef SaaS

Iron Chef is a full-stack restaurant inventory and food waste management application designed to help kitchens better understand and use the ingredients they already have.

The system is being developed to help restaurant teams:

- Track ingredient inventory and quantities
- Monitor expiration dates
- Identify ingredients that are expiring soon
- Track low-stock items
- Reduce unnecessary food waste
- Generate recipe and daily-special ideas based on available inventory
- Increase profits

## Project Background

Iron Chef began as a Systems Analysis and Design college project focused on solving real-world inventory and food waste problems in restaurant kitchens.

The original concept was later developed into a Java desktop prototype. The project is now evolving into a full-stack web application designed for real-world testing and deployment.

## Current Status

🚧 **Active Development**

The project is currently in the prototyping and architecture phase. Multiple development approaches are being evaluated while the database structure, user workflows, and Version 1 MVP are finalized.

## Version 1 Goals

The initial MVP will focus on:

- User authentication
- Restaurant inventory management
- Ingredient quantity and unit tracking
- Expiration tracking and alerts
- Low-stock indicators
- Inventory usage tracking
- Recipe and special suggestions based on available ingredients
- Mobile-friendly kitchen interface

## Development Approach

This project uses AI-assisted development tools for prototyping, boilerplate generation, debugging, and development support.

Product requirements, architecture decisions, database design, testing, feature prioritization, and final implementation decisions are being actively directed and reviewed throughout development.
